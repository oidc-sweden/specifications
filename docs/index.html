Swedish OpenID Connect Specifications

This is the overview page for the Swedish OpenID Connect specifications developed by the Swedish OpenID Connect Working Group. The specifications are currently in "implementer's draft"-mode, and will be released in their 1.0 versions when interoperability testing has been performed.

Take part in the development of the specifications by following the work being made at https://github.com/oidc-sweden/specifications.

The Swedish OpenID Connect Profile

Specification that defines a profile for OpenID Connect for use within the Swedish public and private sectors. It profiles the OpenID Connect protocol to get a baseline security and to facilitate interoperability between relying parties and OpenID providers.

  • Version 1.0 - Draft 02

Attribute Specification for the Swedish OpenID Connect Profile

Specification that defines claims and scopes for the Swedish OpenID Connect profile.

  • Version 1.0 - Draft 02
See also the document How OpenID Connect Claims Map to other Specifications that is a non-normative paper that lists the attributes/claims used in Swedish eID-systems, and states how they map to OpenID Connect claims (defined in "our" specifications or elsewhere).

Authentication Request Parameter Extensions for the Swedish OpenID Connect Profile

Specification that defines authentication request parameter extensions for the Swedish OpenID Connect profile.

  • Version 1.0 - Draft 01

Signature Extension for OpenID Connect

This specification defines an extension to OpenID Connect to facilitate use cases where a Relying Party sends a "Signature Request" to an OpenID Provider. A signature request is an extension of an OpenID Connect authentication request where a "Signature Request" object is passed as a request parameter or a Request Object.

  • Version 1.0 - Draft 02
